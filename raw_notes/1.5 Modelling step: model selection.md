### Заметка: Model Selection Process (Процесс выбора модели)

**Основная концепция**
*   На этапе моделирования наша цель — попробовать разные алгоритмы (например, *logistic regression, decision tree, neural network*), чтобы выбрать из них наилучший. Этот процесс называется **Model Selection Process**.
*   Главная задача — оценить, как модель будет вести себя на **новых, ранее не виданных данных** (имитация реального использования, например, когда мы обучаем модель в июле, а предсказывать ей предстоит данные в августе).

**Разделение данных (Data Splitting)**
Поскольку мы не можем заглянуть в будущее для получения новых данных, мы берем наш существующий датасет и искусственно делим его на непересекающиеся части:
1.  **Training data set (Обучающая выборка)** — основная часть данных (например, 60%), на которой модель `g` обучается. Здесь алгоритм видит и признаки (`X`), и правильные ответы (`y`).
2.  **Validation data set (Валидационная выборка)** — отложенная часть данных (например, 20%), которая используется исключительно для оценки и сравнения моделей. Модель делает предсказания (**predictions**, или `y_hat`), которые мы сравниваем с реальными значениями для вычисления точности (**accuracy**).
3.  **Test data set (Тестовая выборка)** — еще одна отложенная часть данных (например, 20%). Она полностью спрятана до самого конца эксперимента и используется только один раз для финальной проверки.

*Пропорция 60/20/20 не является строгим правилом и может меняться в зависимости от задачи.*

**Проблема: Multiple Comparison Problem**
Зачем нам нужен отдельный *Test data set*? Если мы будем бесконечно тестировать сотни моделей на одном и том же *Validation data set*, какая-то из них может показать идеальную точность просто по случайности. В статистике это называется **multiple comparison problem** (проблема множественных сравнений). 
*   *Пример из видео:* Если предсказывать спам подбрасыванием разных монет (евро, доллары, рубли и т.д.), в какой-то момент одна из монет случайно выдаст последовательность, которая на 100% совпадет с ответами в валидационной выборке. Чтобы убедиться, что нашей лучшей модели "не просто повезло", мы и проводим экстра-проверку на тестовых данных.

**Алгоритм Model Selection Process (6 шагов)**
Процесс выбора модели сводится к следующему алгоритму:
1.  Разделить датасет на 3 части (*Train, Validation, Test*).
2.  Обучить конкретную модель на *Training data set*.
3.  Оценить качество (например, *accuracy*) на *Validation data set*.
4.  Повторить шаги 2 и 3 для других моделей (например, сравнить *logistic regression* и *neural network*).
5.  Выбрать **наилучшую модель** на основе результатов валидации.
6.  Применить эту единственную лучшую модель к *Test data set*, чтобы убедиться, что результаты остаются стабильными и высокими.

**Продвинутый трюк (Оптимизация)**
Чтобы данные из валидационной выборки не "пропадали зря" после того, как мы выбрали лучшую архитектуру модели, можно сделать следующее:
*   Объединить *Training data set* и *Validation data set* в один большой массив данных.
*   Обучить выбранную лучшую модель заново на этом объединенном массиве (она станет работать немного лучше, так как увидела больше примеров).
*   Провести финальную проверку на *Test data set*.
