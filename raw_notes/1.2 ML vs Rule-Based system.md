Сравнение будет производится на примере spam detection. В данном случае, мы будем сравнивать два подхода: машинное обучение (ML) и систему на основе правил (Rule-Based System).

На примере двух писем с изображения

![alt text](image.png)

На которых мы опеределяем что все сообщения с @online.com являются спамом. А также если в теме сообщения есть слова URGENT: tax review и т.д. то это также спам.

### Rule-Based System

В системе на основе правил, мы бы создали набор правил, которые бы определяли, является ли сообщение спамом или нет. Например:
- Если email содержит @online.com, то это спам.
- Если тема содержит слова "tax review", то это спам а также источником письма является @online.com.
- В противном случае, это не спам.

Что мы можем сделать в таком случае? Мы можем написать небольшой python code который будет реализовывать эти правила:

```python
def is_spam(email):
    if "@online.com" in email['from']:
        return True
    if "tax review" in email['subject']:
        return True
    return False
```

И наша программа прекрасно работает пока не приходит другой тип письма с другой темой и другим источником.

![alt text](image-1.png)

В таком случае нам надо будет добавлять новые правила, например:
- Если в теле письма указываем слово "deposit" то это спам

```python
def is_spam(email):
    if "@online.com" in email['from']:
        return True
    if "tax review" in email['subject'] and "@online.com" in email['from']:
        return True
    if "deposit" in email['body']:
        return True
    return False
```

И все продолжает работать нормально снова до того момента пока мы не получаем письмо о том что кто то заплатил нам 100$ но сейчас не нуждается в наших услсугах и хочет вернуть деньги. В таком случае в письме будет фигурировать слово "deposit" и наша система будет ошибочно классифицировать его как спам.

![alt text](image-2.png)

Мы снова добавляем новые правила, например:
- если отправитель test.com и в теме письма есть слово "deposit" то это спам
- если длина текста письма больше 100 слов и в теме письма есть слово "deposit" то это спам

```python
def is_spam(email):
    if "@online.com" in email['from']:
        return True
    if "tax review" in email['subject'] and "@online.com" in email['from']:
        return True
    if "deposit" in email['body'] and "test.com" in email['from']:
        return True
    if "deposit" in email['subject'] and len(email['body'].split()) > 100:
        return True
    return False
```

И это будет продолжать бесконечно и каждый раз надо будет держать скрипт (блок кода) обновленным и добавлять новые правила. И в итоге мы получаем огромный код который сложно поддерживать и который может работать неэффективно.

### Machine Learning

В случае с машинным обучением, мы бы собрали большой набор данных писем, которые были помечены как спам или не спам. Затем мы бы использовали алгоритмы машинного обучения для обучения модели на этих данных. Модель бы научилась распознавать паттерны в данных и могла бы делать предсказания о том, является ли новое письмо спамом или нет.

Пошаговый процесс:
- Получить данные
- Определить и рассчитать features 
- Обучить и использовать модель для предсказания

Features для писем могут включать:
- Длина тела письма
- Длина темы письма
- Отправитель
- Домен отправителя
- Содержит ли письмо слово "deposit"

Если вы обратили внимание то features это теже самые rules которые мы определили в rule-based system. И это является отличной практикой для определения features для модели. Но в случае с машинным обучением, мы не ограничены только этими features и можем использовать гораздо больше информации для обучения модели. И модель будет учиться на данных и находить оптимальные параметры для классификации писем как спам или не спам, а не полагаться на жестко заданные правила.

Binary fetures - это features которые могут принимать только два значения, например:
- Содержит ли письмо слово "deposit" (да/нет)
- Является ли отправитель доменом "online.com" (да/нет)

Что мы получим как результат при использовании модели

![alt text](image-3.png)

Каждая 1 или 0 показывает верно ли feature для указанного письма. И модель будет использовать эти features для предсказания того, является ли письмо спамом или нет. И если мы получаем новое письмо, то модель будет использовать эти features для предсказания его класса (спам или не спам) без необходимости добавлять новые правила каждый раз, когда приходит новый тип письма.

После обучения модель будет отдавать вероятность того что письмо является спамом. Между результатов предсказания и конечным результатом spam или не спам, мы можем установить порог, например 0.5. Если вероятность того что письмо является спамом больше 0.5, то мы классифицируем его как спам, иначе как не спам.

Таким образом, машинное обучение позволяет нам создавать более гибкие и адаптивные модели, которые могут обрабатывать большое количество данных и находить сложные паттерны, в то время как системы на основе правил могут стать громоздкими и неэффективными при обработке большого количества данных и разнообразных случаев.
