# Model Selection Process

Цель этапа моделирования — попробовать разные алгоритмы (logistic regression, decision tree, neural network) и выбрать наилучший. Ключевой критерий: как модель поведёт себя на **новых, ранее не виданных данных**.

## Разделение данных (Data Splitting)

Датасет делится на три непересекающиеся части:

| Часть | Доля | Назначение |
|---|---|---|
| **Training set** | ~60% | Обучение модели (видит `X` и `y`) |
| **Validation set** | ~20% | Сравнение и отбор моделей |
| **Test set** | ~20% | Финальная проверка один раз |

> [!tip] Пропорция 60/20/20 — не строгое правило, варьируется под задачу.

## Зачем нужен отдельный Test set?

> [!warning] Multiple Comparison Problem
> При многократном тестировании сотен моделей на одном Validation set какая-то из них покажет отличный результат **случайно**. Test set защищает от этого эффекта — он используется только один раз, в самом конце.

[!example] Аналогия: если подбрасывать достаточно монет (евро, доллары, рубли...), одна случайно выдаст последовательность, совпадающую с ответами валидационной выборки на 100%.

## Алгоритм Model Selection (6 шагов)

1. Разделить датасет на Train / Validation / Test
2. Обучить модель на Training set
3. Оценить accuracy на Validation set
4. Повторить шаги 2–3 для других моделей
5. Выбрать **наилучшую модель** по результатам валидации
6. Применить её к Test set для финальной проверки

## Продвинутый трюк: объединение Train + Validation

После выбора архитектуры модели:

1. Объединить Training set и Validation set
2. Переобучить выбранную модель на объединённых данных (больше примеров → чуть лучше качество)
3. Провести финальную проверку на Test set

> [!summary] Итог
> Data Splitting — основной инструмент честной оценки модели. Validation set используется для выбора, Test set — только для финального подтверждения.
