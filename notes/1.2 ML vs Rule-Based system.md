# 1.2 ML vs Rule-Based System

## Контекст: задача Spam Detection

Сравниваем два подхода на примере классификации электронных писем как **спам / не спам**.

---

## Rule-Based System (Система на правилах)

Подход: вручную описываем условия, при которых письмо считается спамом.

```python
def is_spam(email):
    if "@online.com" in email['from']:
        return True
    if "tax review" in email['subject']:
        return True
    return False
```

### Проблема: правила накапливаются

Каждый новый тип спама требует нового правила:

```python
def is_spam(email):
    if "@online.com" in email['from']:
        return True
    if "tax review" in email['subject'] and "@online.com" in email['from']:
        return True
    if "deposit" in email['body'] and "test.com" in email['from']:
        return True
    if "deposit" in email['subject'] and len(email['body'].split()) > 100:
        return True
    return False
```

> [!warning] Главная проблема Rule-Based
> Код разрастается бесконечно. При этом новые правила легко конфликтуют со старыми — как в случае письма о возврате 100$, где слово "deposit" присутствует, но письмо не является спамом.

---

## Machine Learning подход

Вместо ручного описания правил — **обучаем модель на данных**.

### Процесс

```
1. Собрать данные (письма, размеченные как спам / не спам)
        ↓
2. Вычислить features для каждого письма
        ↓
3. Обучить модель на этих данных
        ↓
4. Использовать модель для предсказания новых писем
```

### Features для писем

| Feature | Тип | Пример |
|---|---|---|
| Длина тела письма | Числовой | 250 слов |
| Длина темы | Числовой | 5 слов |
| Домен отправителя — `online.com` | Бинарный | 1 / 0 |
| Содержит слово "deposit" | Бинарный | 1 / 0 |
| Содержит слово "tax review" | Бинарный | 1 / 0 |

> [!tip] Связь с правилами
> Features в ML — это те же самые условия, что мы вручную описывали в rule-based системе. Разница в том, что **модель сама определяет их вес**, а не программист задаёт жёсткие пороги.

### Binary Features

**Binary features** — признаки, принимающие только два значения (0 или 1):
- Содержит ли письмо слово "deposit"? → `1` (да) / `0` (нет)
- Отправитель с домена `online.com`? → `1` (да) / `0` (нет)

### Результат предсказания

Модель возвращает **вероятность** того, что письмо является спамом.

```
Вероятность спама: 0.87  →  [порог 0.5]  →  SPAM
Вероятность спама: 0.12  →  [порог 0.5]  →  NOT SPAM
```

Порог можно настраивать под конкретную задачу.

---

## Сравнение подходов

| Критерий | Rule-Based | Machine Learning |
|---|---|---|
| Сложность поддержки | Растёт с каждым новым случаем | Постоянная — достаточно переобучить модель |
| Адаптация к новым данным | Ручное обновление правил | Автоматически при переобучении |
| Прозрачность | Высокая — правила явно заданы | Ниже — модель — "чёрный ящик" |
| Количество данных | Не требует данных | Требует размеченного датасета |
| Гибкость | Ограничена | Высокая |

---

## Итог

> [!summary]
> Rule-based системы хорошо работают для простых и стабильных задач, но становятся неуправляемыми по мере роста разнообразия случаев. ML позволяет модели **самой находить оптимальные параметры** на основе данных — без необходимости вручную обновлять логику при каждом новом сценарии.
