# 1.3 Supervised Machine Learning (Обучение с учителем)

## Основная концепция

**Supervised ML** — обучение алгоритма на примерах, для которых уже известен правильный ответ.

> [!tip] Аналогия
> Модель — «ученик», мы — «учитель». Показываем данные с ответами → модель извлекает закономерности (**patterns**) → применяет их к новым данным (**generalize**).

---

## Формализация задачи

| Обозначение | Название | Описание |
|---|---|---|
| `X` | Feature matrix | Таблица: строки — объекты, столбцы — признаки |
| `y` | Target variable | Вектор с правильными ответами |
| `g` | Модель (функция) | То, что мы хотим найти |

**Цель обучения:** найти функцию `g` такую, что:

```
g(X) ≈ y
```

Процесс подбора параметров `g` называется **training** (обучением).

---

## Типы задач Supervised ML

### 1. Regression (Регрессия)

`g` возвращает **число**.

> [!example]
> Предсказание цены автомобиля по пробегу, года выпуска, марки.
> Предсказание стоимости дома по площади и удалённости от центра.

### 2. Classification (Классификация)

`g` возвращает **категорию (класс)**.

**Binary classification** — два класса (`0` / `1`):
- Спам или не спам
- Модель возвращает вероятность от 0 до 1

> [!important]
> Бинарная классификация — самый распространённый тип задач ML на практике.

**Multi-class classification** — три и более класса:
- Что на изображении: машина, собака или кошка?

### 3. Ranking (Ранжирование)

Модель присваивает каждому элементу **score** (оценку релевантности) и сортирует по убыванию.

> [!example]
> - Поисковые системы (Google)
> - Рекомендательные системы (маркетплейсы, стриминг)

---

## Итог

```
Тип target variable → тип задачи
        ↓
  Число    → Regression
  Категория → Classification (Binary / Multi-class)
  Порядок  → Ranking
```
